## 🎯 Project Overview

This project implements a **Mini Relational Algebra Engine (MMA)** built from scratch in Java 17. The engine acts as a lightweight database processor that parses, translates, and executes core relational algebra queries (selection, projection, and equi-join) over standard CSV files.

The core architecture completely avoids imperative iteration loops, embracing **functional programming models** through Java Streams, pure functions, and recursive pipeline evaluations.

### **✨ Key Architectural Pillars:**
* **🧩 Object-Relational Memory Mapping:** Implements database models natively using decoupled components (Tables, Rows, Attributes) where rows are stochastically mapped using dynamic attribute dictionaries.
* **⚡ Functional Pipeline Execution:** Processes data queries dynamically using a compositional interface where operators (Scan, Select, Project, Join) act as pure immutable functions transformation layers.
* **🗜️ Abstract Syntax Tree (AST) Compiler:** Features a custom lexical scanner (AlgTranslator) capable of compiling raw iterative logic scripts (.alg) into standardized relational execution blueprints (.json).
* **📦 Enterprise Build Standard:** Integrated with Apache Maven Wrapper, enforcing strict Google Code Style definitions via Checkstyle automated validation hooks.

## 🔧 Getting Started & Usage Guide

### 🛠️ Execution Requirements
Ensure Java Development Kit (JDK) 17 and Maven are configured in your development ecosystem. Build the final standalone execution archive using the Maven wrapper:
./mvnw clean package

### 🚀 Running an Algebra Plan Execution (.json)
To evaluate an algebraic tree plan from a structured JSON file and dump output tables into localized target directories, invoke the primary archive:
java -jar target/collex-jar-with-dependencies.jar test/reqProjSel.json

### 🔄 Compiling Script Configurations (.alg)
To translate a domain-specific custom text configuration into a structural relational query JSON blueprint, pass the script parameter:
java -jar target/collex-jar-with-dependencies.jar test/reqProjSel.alg

## 📊 Technical Architecture & Design Q&A

### Which frameworks were selected and why?
* Google Gson: Fast serialization and dynamic construction of recursive plan nodes from variable input streams.
* JUnit 5: Decoupled unit testing assertion hooks verifying relational boundaries.
* JaCoCo: Automatic test-coverage inspection monitoring mutation branch quality metrics.

### Why is JSON highly appropriate for representing execution trees?
* Positives: Highly standardized, readable formatting that naturally matches the hierarchical nested structure of relational query execution steps.
* Negatives: High structural verbosity for manual authoring, semantic structural integrity validation can only be achieved at application execution time.

### What are the benefits of functional over imperative approaches here?
Functional implementations create declarative execution schemas where operators are entirely composable. Relational operations behave as pure mathematical expressions transforming an immutable dataset into a new isolated database table result, significantly mitigating state side-effects.
